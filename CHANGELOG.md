# 变更日志

本文件记录项目的正式变更历史。
日常调试过程、阶段性排障结论和零散开发记录请写入 `技术日志.md`。

## 使用说明
- 开发中的变更先写入 `未发布`
- 形成稳定版本时，将内容整理到对应日期条目下
- 建议每条记录带上模块前缀，例如：`[Driver]`、`[Device]`、`[Controller]`
- 每条尽量描述“做了什么”和“影响了什么”

### 变更
- [Docs] 调整 README 中 ZDT 电机总览描述，明确为力矩环与 EmmX CAN 扩展帧协议。
- [Task] LCD 状态数据改为每 100 ms 在任务态提交，运行期由 DMA 状态机重绘变化字段，TIM5 中断不执行 SPI 传输。
- [Device] LCD 源文件补充与现有设备驱动一致的文件头、模块分区和公开接口注释。
- [Device] LCD 驱动改为固定 ST7789 屏幕的通用绘制接口，加入图元、数值、字体、旋转、背光和固定容量异步队列。
- [Interaction] 双机械臂状态页迁移到独立 LCD 页面模块，设备驱动不再依赖机械臂结构体。
- [Driver] SPI 增加按句柄分发 TX 完成/错误事件，LCD 不再维护全局活动实例。

## 2026-07-08

### 新增
- [Device] ZDT 电机驱动新增位置速度、速度、力矩三种控制方式，并补充位置/电流查询与反馈处理。
- [Driver] CAN 底层新增扩展帧发送接口 `CAN_Send_Extended_Data()`，供 ZDT 驱动使用。
- [Task] 任务初始化流程新增机械臂 CAN 全局过滤配置，用于接收 ZDT 扩展帧回包。

### 变更
- [Controller] 机械臂 CAN 回调改为按扩展帧设备号分发 ZDT 回包，并向驱动传递完整 CAN 帧。
- [Device] `Class_ZDT_Motor::Init()` 扩展为支持控制方式、限扭/限流/限速等参数配置，并完善使能边沿与在线状态管理。

### 修复
- [Device] 完善 ZDT 扩展帧回包解析路径，使位置、电流和在线状态能够正确更新。
